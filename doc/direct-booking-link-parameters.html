Racehall Direct Booking

Direct Booking Link Parameters for racehall.com

This handoff defines the exact URL contract for direct booking links. The handler can run from any normal public frontend URL, as long as the required query parameters are present and correctly formatted.

Canonical Example
https://racehall.com/?rh_booking=1&product_id=123&date=2026-04-15&time=18:30&location=Aarhus&adults=6&children=2&twin=0&quantity=8
Core Rule

rh_booking should use the default value 1. If quantity is included, it must equal adults + children + twin.

Required Parameters

Parameter Required Purpose Example
rh_booking Yes Turns the direct booking handler on. The default value should be 1. 1
product_id Yes WooCommerce product ID that should be added to cart. 123
date Yes Booking date in strict YYYY-MM-DD format. 2026-04-15
time Yes Booking time in strict HH:MM format. 18:30
location Yes Booking location. If the Woo product already has a configured location, that value is used and this parameter is ignored if different. If the product has no configured location, this parameter is required. URL-encode special characters. Aarhus
adults Yes Number of adult participants. 6
children Yes Number of child participants. 2
twin Yes Number of twin participants. 0
quantity Optional Total participants. If provided, it must equal adults + children + twin. If omitted, the handler derives it from the participant split. 8
Recommended path example: https://racehall.com/?rh_booking=1.... The handler can also run on other normal public frontend paths.

Validation Rules

Format: Use exact date and time formats only.
Totals: If quantity is provided, it must equal adults + children + twin.
Location: Product configuration takes precedence over the URL parameter.
Slot lookup: If the slot cannot be resolved upstream, the booking is rejected.

Use This Exact Style

Do not send localized date strings or full datetime values. Including quantity is recommended for determinism, but it is not required.
