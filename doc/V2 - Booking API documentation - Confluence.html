Security

The API is secured with authentication using a bearer token. Clients should obtain an access token by sending a POST request to the /auth/{clientkey}/publicbooking endpoint with their username and password in the request body. The response includes a bearer access token that should be included in the Authorization header of subsequent requests. The access token has an expiration time of 3600 seconds and is included in a list of valid client keys for added security.

Error response(s)

400

A 400 (Bad request) response will return a ErrorResponse dto

public class ErrorResponse { public ExceptionKind Kind { get; set; } public string Message { get; set; } }

ExceptionKind

public enum ExceptionKind { Unknown, Validation, Database, Business, ActionRequired, }

401

A 401 (Unauthorized) response indicate either in passing a expired or wrong JWT.

Also serving a ClientKey in the URL parameters that is not included in the JWT claims will also results in a 401.
